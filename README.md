## 🤝 기여하기

새로운 사이트를 알고 계신가요? Pull Request로 자유롭게 기여해주세요!

### 기여 전 확인사항

- [ ] 이미 목록에 있는 사이트인지 확인
- [ ] 실제로 연봉 정보를 제공하는 사이트인지 확인
- [ ] 사이트가 현재 정상 운영 중인지 확인

### 목록에 넣기 전 참고

공개 범위, 저작권, 개인정보를 기준으로 링크를 골라 주세요. 이 레포는 **공개적으로 참고할 수 있는 연봉·보상 관련 출처**를 모읍니다. 아래는 PR·이슈로 링크를 제안할 때 함께 점검하면 좋은 기준입니다. (법률 자문이 아니며, 최종 판단은 기여자·운영자 각자의 책임입니다.)

**추가를 피하거나, 링크만 하고 원문을 복제하지 말아야 할 경우**


1. **비공개·회원 전용** — 로그인 없이는 연봉·통계 본문을 볼 수 없는 경우 (초대 전용, 유료 구독 전용 등).
2. **민감한 개인정보** — 실명, 연락처, 주민등록번호 등 식별·연락 가능한 개인 단위 정보가 노출되거나, 그런 형태로 수집·공유되는 출처.
3. **이용약관·저작권·사이트 정책상 재배포 금지** — `robots.txt`·약관·푸터 등에 **공유 금지**, **내부용**, **무단 전재 금지** 등이 명시된 경우. 해당 사이트 정책을 우선합니다.


**비교적 목록에 넣기 적합한 경우 (여전히 약관·법령은 각 사이트를 확인)**

- 페이지에 **공유하기·임베드·인용 안내** 등 공개 배포를 전제로 한 기능이 있는 경우.
- **오픈 데이터**·정부·공공기관 등 명시적으로 재이용이 허용된 데이터인 경우.
- **별도의 퍼감·재배포 금지** 문구가 없고, 로그인 없이 요약·통계를 열람할 수 있는 경우.

위 기준에 맞지 않는 출처는 PR에서 거절되거나 링크만 유지·삭제될 수 있습니다.

### 기여 방법

1. 이 레포지토리를 **Fork**
2. 새 브랜치 생성: `git checkout -b add/사이트명`
3. 아래 템플릿 형식에 맞게 항목 추가
4. 커밋: `git commit -m "add: 사이트명 추가"`
5. Push 후 **Pull Request** 생성

> 사이트 제안만 하고 싶다면 [Issue](../../issues/new)를 열어주세요.  
> ⭐ 추천 여부는 관리자가 직접 판단하며, PR에서 추천 요청도 가능합니다.

### 추가 템플릿

```markdown
|    | [사이트 이름](https://example.com) | 연도 | 대상 | 한 줄 설명 |
```

**연도** 작성 기준: `실시간` / `2025` / `2024~` / `비공식`  
**대상** 작성 기준: `전 직군` / `개발자` / `프론트엔드` / `JS 개발자` / `React 개발자` / `디자이너` 등
